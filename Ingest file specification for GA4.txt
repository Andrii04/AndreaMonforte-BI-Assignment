Ingest file specification for GA4 raw CSV

	Filename convention:
		-Accepted filename: ga4_public_dataset.csv

	File format and encoding:
		-Format: CSV (comma-separated values)
		-Delimiter: , (comma)
		-Quote character: " (double quote)
		-Escape for quotes: "" (standard CSV escaping)
		-Encoding: UTF-8
	
	Header and schema rules:
		-Evader: the first line MUST contain column names exactly as specified (case-sensitive).
		-Required columns: (must exist and not be NULL)
			event_name (STRING)
			event_date (INTEGER) - format YYYYMMDD
			event_timestamp (INTEGER) - epoch milliseconds preferred
			user_pseudo_id (FLOAT)
			event_params (RECORD) - containing the following fields:
				key (STRING)
				value (RECORD) - containing the following fields:
					double_value (FLOAT)
					float_value (STRING)
					int_value (INTEGER)
					string_value (STRING)
			device (RECORD) - containing:
				web_info (RECORD) - containing the following fields:
					time_zone_offset_seconds (STRING)
					language (STRING)
					operating_system (STRING)
					mobile_os_hardware_model (STRING)
					mobile_marketing_name (STRING)
					is_limited_ad_tracking (BOOLEAN)
					advertising_id (STRING)
					category (STRING)
					operating_system_version (STRING)
					mobile_model_name (STRING)
					vendor_id (STRING)
					mobile_brand_name (STRING)

	Data types and value formats:
		-STRING fields: must be quoted when containing commas or newlines. Avoid unescaped newlines.
		-Numeric fields: plain digits only; no thousands separators or currency symbols.
